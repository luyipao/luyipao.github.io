make语法之一

本文最后更新于：1 年前

简介

make 是一个经由读取 “makefile” 的文件的自动化构建软件。make 像一个工具来处理 Makefile 里的规则。

规则

target(目标)

这里的 target 是指规则的目标，是一条规则的核心，一条规则可以没有 prerequisites，也可以没有 commands 但绝对不能没有 target。这个 target 往往是文件名，指明 Make 命令所要构建的对象，也可以是操作名，这称为 phony target 例如 clean 。但是如果本身就有 clean 文件，运行make clean会报错'clean' is up to date.。为了避免这种情况发生可以用.PHONY声明 clean 是 “伪目标”

这样clean会绕过检查 clean 文件是否存在.

利用make target运行规则，但如果 target 为空将默认执行第一条规则，因此人们常常将第一条规则命名为 all，单独列出来。

prerequisites(前置条件)

利用 make + target 运行规则时首先检查prerequisites是否存在，然后检查 target，最后才会运行命令。

但是如果 result 的prerequisites source 已经存在则直接运行命令，否者需要一个target为 source 的规则生成一个 source 文件。

这里相当于检测到 source 规则后先运行了一个make source命令。

command(命令)

命令（commands）表示如何更新目标文件，由一行或多行的Shell命令组成。它是构建"目标"的具体指令，它的运行结果通常就是生成目标文件。

命令前必须有一个 Tab 键, 可以用.RECIPEPREFIX声明更改 Tab 键。

总感觉没什么用。每行分别在单独的 shell 中执行，各行 command 间没有继承关系。但是可以通过将多个命令写在同一行建立继承关系。

语法

注释

用#在 Make file 中添加注释.

echoing(回声)

正常情况下，make会打印每条命令，然后再执行，这就叫做回声（echoing）。

运行make test可得到

如果在#前加上@就可以关闭回声.

由于在构建过程中，需要了解当前在执行哪条命令，所以通常只在注释和纯显示的echo命令前面加上@。
